## Creating Knowledge Bases

<Steps>
  <Step title="Create New Knowledge Base">
    Navigate to **Knowledge** in the CloudThinker console and click **"Create Knowledge Base"**
  </Step>
  <Step title="Choose Knowledge Base Type">
    Select between **Document Upload** or **Web Crawler** based on your content source
  </Step>
  <Step title="Configure Basic Settings">
    - **Name**: Descriptive title (e.g., "AWS Operations Procedures")
    - **Description**: Brief overview of the knowledge base purpose
    - **Tags**: Relevant keywords for organization and discovery
  </Step>
  <Step title="Set Access & Usage Permissions">
    - **Access Level**: Private, Team, or Workspace sharing
    - **Usage Mode**: Manual (#kb command) or Always (automatic inclusion)
    - **Agent Access**: Which agents can access this knowledge base
  </Step>
  <Step title="Add Content">
    Upload documents or configure web crawling based on your knowledge base type
  </Step>
</Steps>

## Web Crawling & Integration

### Website Crawling Setup

<Steps>
  <Step title="Configure Crawler Source">
    **Website Settings:**
    - **Start URL**: Primary webpage to begin crawling
    - **Domain Scope**: Restrict crawling to specific domains
    - **Depth Limit**: Maximum number of links to follow (1-10)
    - **Page Limit**: Maximum pages to crawl (100-10,000)
  </Step>
  
  <Step title="Content Filtering">
    **Crawling Rules:**
    - **Include Patterns**: URL patterns to crawl (`/docs/*`, `/wiki/*`)
    - **Exclude Patterns**: URLs to avoid (`/admin/*`, `/private/*`)
    - **Content Types**: HTML, PDF, and text content only
    - **Language Filter**: Specify preferred languages
  </Step>
  
  <Step title="Crawling Schedule">
    **Update Frequency:**
    - **One-time**: Single crawl operation
    - **Daily**: Update content every 24 hours
    - **Weekly**: Weekly content refresh
    - **Manual**: On-demand crawling only
  </Step>
  
  <Step title="Authentication & Access">
    **Security Configuration:**
    - **Public Access**: No authentication required
    - **Basic Auth**: Username/password authentication
    - **API Keys**: Custom header authentication
    - **Cookie-based**: Session-based access
  </Step>
</Steps>

## Troubleshooting & Support

### Common Issues & Solutions

<Accordion title="🔍 Search Issues">
  **Problem**: Agents can't find relevant information in knowledge base
  
  **Solutions:**
  1. **Improve Document Structure:**
     - Add clear headings and sections
     - Include relevant keywords in document titles
     - Create comprehensive document summaries
  
  2. **Optimize Search Terms:**
     - Use natural language in documents
     - Include synonyms and alternative terms
     - Add FAQ sections with common questions
  
  3. **Check Agent Configuration:**
     - Verify agent has access to knowledge base
     - Confirm usage mode (Always vs Manual)
     - Test with specific `#kb` commands
</Accordion>

<Accordion title="📤 Upload Issues">
  **Problem**: Documents failing to upload or process
  
  **Solutions:**
  1. **File Format Issues:**
     - Verify file type is supported
     - Check file size (max 50MB per file)
     - Ensure documents are not password-protected
  
  2. **Content Processing:**
     - For PDFs: Ensure text is selectable (not image-only)
     - For Office docs: Save in modern formats (.docx, .pptx)
     - For images: Include alt text and descriptions
  
  3. **System Resources:**
     - Upload during off-peak hours for large files
     - Use batch upload for multiple documents
     - Monitor upload progress and retry if needed
</Accordion>

<Accordion title="🕷️ Web Crawling Issues">
  **Problem**: Web crawler not accessing target sites
  
  **Solutions:**
  1. **Access Configuration:**
     - Verify authentication credentials
     - Check robots.txt compliance
     - Confirm domain whitelist settings
  
  2. **Crawling Parameters:**
     - Adjust depth and page limits
     - Review include/exclude patterns
     - Test with simple single-page crawls first
  
  3. **Content Quality:**
     - Monitor crawling success rates
     - Review content extraction quality
     - Adjust content filters as needed
</Accordion>
